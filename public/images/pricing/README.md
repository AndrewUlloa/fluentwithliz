# Pricing Page Images

Place the following images in this directory (`/public/images/pricing/`):

## Required Images

1. **hero-pricing.jpg** (1200x675px recommended)
   - Hero image at the top of the pricing page
   - Should represent investment, learning, or value
   - Full-width banner

2. **group-learning.jpg** (1200x675px recommended)
   - Image showing group English classes
   - Students learning together, interactive classroom
   - Used in "Why Group Classes?" section

3. **module-1.jpg** (400x300px recommended)
   - Image representing Module 1: Foundation
   - Could be beginner concepts, starting point
   - Used in module structure grid

4. **module-2.jpg** (400x300px recommended)
   - Image representing Module 2: Building Skills
   - Could be intermediate learning, progress
   - Used in module structure grid

5. **module-3.jpg** (400x300px recommended)
   - Image representing Module 3: Mastery
   - Could be advanced concepts, achievement
   - Used in module structure grid

6. **adult-learners.jpg** (1200x675px recommended)
   - Image showing motivated adult English learners
   - Adults in class, professional learning environment
   - Used in "For Motivated Adult Learners" section

7. **investment-future.jpg** (1200x675px recommended)
   - Image representing investment in future
   - Could be success, transformation, or opportunity
   - Used in "An Investment in Your Future" section

8. **cta-background.jpg** (1200x400px recommended)
   - Background image for CTA section
   - Subtle, low-opacity background
   - Used as background in final CTA section

## Image Guidelines

- **Format**: JPG or WebP (WebP preferred for better compression)
- **Aspect Ratios**: Maintain the recommended aspect ratios for best display
- **Optimization**: Images will be automatically optimized by Next.js Image component
- **Alt Text**: Already included in the component, but ensure images match the descriptions
- **File Names**: Use exact file names as listed above (case-sensitive)

## Image Placeholders

Until images are added, the page will show loading skeletons. The images will gracefully handle missing files with error states.
